# CRM 客户关系管理系统 (Qt 6.8 + SQLite)

## 1. 项目简介
本系统是一款为销售团队打造的轻量级客户管理与跟进平台，采用 Qt 6.8 + C++ 开发，底层使用 SQLite 数据库实现单文件本地存储与无缝交接。

## 2. 核心功能
- **智能权限管理**：根据角色自动过滤数据视图。
- **全生命周期客户管理**：从线索录入到跟进记录全覆盖。
- **动态时间线**：通过 `QPainter` 自定义绘制客户沟通历史。
- **公海池机制**：支持客户认领与自动回收。
- **人事管理**：支持员工部门装接、权限变更及一键交接离职资产。

## 3. 角色权限架构
| 角色 | 权限范围 |
| :--- | :--- |
| **销售 (Sales)** | 仅能查看/管理自己名下的客户，提交跟进记录，认领公海池客户。 |
| **经理 (Manager)** | 可查看所属部门全体销售的客户列表及跟进概况。 |
| **管理员 (Admin)** | 拥有上帝视角，全盘查看、增删改查、分配员工部门、一键交接资产。 |

## 4. 技术栈说明
- **开发环境**: Qt 6.8, C++20
- **数据库**: SQLite (本地单文件存储)
- **核心模式**: 面向接口编程 (ICustomerRepository)，实现前后端解耦。

## 5. 项目结构
- `/src/models/`: 数据模型层 (`User`, `Customer`, `FollowRecord`)
- `/src/interfaces/`: 仓储接口层 (`ICustomerRepository.h`)
- `/src/repositories/`: 实现层 (SQLite 驱动实现)
- `/src/ui/`: 前端界面 (表格、时间线组件)

---
## 6. 开发约定
- **所有数据交互必须经过 ICustomerRepository 接口**，严禁前端直接操作数据库。
- **所有实体类使用 const 引用传参**，提升系统性能。
